# Contributing Guidelines

Thank you for considering contributing to this project.

## How to Contribute

1. Fork the repository
2. Create a new branch  

feature/<short-description>

or  

bugfix/<short-description>

3. Make your changes with clear, readable code
4. Add comments or documentation where necessary
5. Commit with a clear message
6. Submit a Pull Request describing:
- What was changed
- Why it was changed
- How it was tested

## Code Standards

- Follow existing project structure
- Keep functions modular and readable
- Avoid adding unnecessary dependencies
- Ensure notebooks and scripts run without errors

## Reporting Issues

Please use the provided issue templates for:
- Bug reports
- Feature requests

This helps keep discussions clear and actionable.
