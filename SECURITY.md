# Security Policy

## Supported Versions

This project is primarily intended for learning, demonstration, and portfolio purposes.
Only the latest version available on the `main` branch is supported.

## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly.

❌ Do NOT open a public GitHub issue for security-related concerns.

✅ Instead, report the issue privately by:
- Contacting the repository owner via GitHub profile  
OR
- Sending details through a private communication channel if provided

Please include:
- A clear description of the vulnerability
- Steps to reproduce the issue (if applicable)
- Potential impact or risk assessment

## Disclosure Policy

Once a vulnerability is reported:
- It will be reviewed as soon as possible
- Reasonable efforts will be made to address valid security concerns
- Public disclosure will occur only after a fix or mitigation is available, if applicable

## Scope

This security policy applies to:
- Source code
- ML pipelines
- Data handling logic
- Dependency usage defined in this repository

It does not cover:
- Third-party datasets
- External services or platforms used for experimentation

---

Thank you for helping keep this project secure and responsible.
